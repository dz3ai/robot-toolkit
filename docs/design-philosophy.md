# robot-toolkit 设计哲学

## 核心理念

robot-toolkit 的设计遵循以下原则：

### 1. 正确性优先 (Correctness First)

机器人运动学和动力学计算涉及人身安全——错误的关节力矩或奇异位形可能导致设备损坏或人员伤害。因此：

- 每个算法实现都有解析验证（如单摆重力力矩与理论公式 `mgL sin(θ)` 的对比）
- 动力学往返一致性检验：逆动力学 + 正动力学应恢复原始加速度
- IK 解算结果通过正向运动学回代验证位置精度 < 0.1 mm
- 所有数值计算使用 `numpy.float64`，避免单精度累积误差

### 2. 分层架构 (Layered Architecture)

```
┌─────────────────────────────────────┐
│  应用层: 可视化、轨迹规划、控制     │
├─────────────────────────────────────┤
│  算法层: IK、动力学、URDF 解析      │
├─────────────────────────────────────┤
│  核心层: DH 参数、齐次变换矩阵     │
├─────────────────────────────────────┤
│  加速层: C++ 扩展 (pybind11)        │
└─────────────────────────────────────┘
```

每一层可独立使用，不依赖上层的存在。用户可以只用 `dh_transform()` 计算单个变换矩阵，也可以用完整的 `ik_solve()` 进行逆运动学求解。

### 3. Python 优先、C++ 加速 (Python-First, C++ Accelerated)

- 所有算法首先用纯 Python 实现，确保可读性和可调试性
- C++ 扩展使用 pybind11 封装，通过 `setup.py build_ext --inplace` 编译
- 运行时自动检测：C++ 可用时透明加速，不可用时回退到 Python
- 性能数据完全可复现：每个测试套件都包含 benchmark

### 4. 最小依赖 (Minimal Dependencies)

- 核心依赖仅 `numpy >= 1.24.0`
- 可视化可选依赖 `matplotlib >= 3.7.0`
- C++ 编译可选依赖 `pybind11`
- 不依赖 ROS、Eigen 或其他重型框架

## 数据表示约定

| 量纲 | 单位 | 说明 |
|------|------|------|
| 长度 | 米 (m) | DH 参数中的 a, d，末端位置 |
| 角度 | 弧度 (rad) | 关节角、DH 参数中的 alpha |
| 时间 | 秒 (s) | 速度、加速度、轨迹时间 |
| 质量 | 千克 (kg) | 连杆质量 |
| 惯量 | kg·m² | 转动惯量张量 |
| 力矩 | N·m | 关节力矩 |
| 力 | N | 力、重力 |

坐标系约定：标准 DH (Denavit-Hartenberg) 约定。
变换矩阵：4×4 齐次变换矩阵，旋转矩阵按列优先存储。

## 错误处理策略

- 不使用异常处理数值问题，而是使用稳健的数值方法（阻尼最小二乘、条件数自适应阻尼）
- Jacobian 奇异时自动增加阻尼因子，而非报错
- IK 收敛失败返回 `(False, q_best, iterations, errors)`，由调用者决策
- 动力学惯性矩阵奇异时添加正则化项 `1e-6 * I`
